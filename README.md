# 媒体嗅探下载器

一个基于 Chrome Manifest V3 的浏览器扩展，用来在当前页面中嗅探已经实际加载过的媒体资源，并按分类展示、预览、下载。

它更接近“当前页媒体资源提取工具”，不是全站爬虫。

## 这个项目能做什么

- 自动监听当前标签页里的媒体请求和页面内资源引用
- 识别并归类音频、视频、图片和其他可下载资源
- 在扩展弹窗中查看资源列表并一键下载
- 把资源面板固定到网页右上角，边浏览边下载
- 对部分直接下载容易失败的资源，尝试通过页面上下文重新抓取后再下载，提高成功率

## 它的工作方式

项目同时用了两种手段：

1. 后台通过 `chrome.webRequest` 监听网络响应
2. 内容脚本在页面里 hook `fetch`、`XMLHttpRequest`、`audio/video/src`、DOM 变化

只有“页面真实出现过、真实请求过”的资源才有机会被捕获。

这意味着：

- 当前页面没触发加载的资源，它抓不到
- 它不会主动遍历整个网站
- 它不是 DRM 破解工具
- 它不是去水印工具

## 站点理解

### 爱给网 `aigei.com`

代码里对爱给网做了一个很轻量的站点策略：

- 除常见音频、视频、图片外
- 还会把 `zip`、`rar` 识别为可抓取资源

所以更准确的说法是：

- 爱给网当前页面里实际加载或请求到的资源，有机会被抓到
- 不是“自动把爱给全站所有资源都拿下来”

### 即梦 `jimeng.jianying.com`

当前代码里没有对 `jimeng.jianying.com` 做专门适配，也没有“去水印”逻辑。

它能做到的是：

- 如果页面里实际请求了图片地址，这个扩展可能会嗅探到
- 如果页面拿到的是原图链接，你可能能直接下载原图

它做不到的是：

- 主动去除服务端返回图片中的水印
- 破解加密、鉴权或 DRM

## 安装方式

1. 打开 Chrome 或 Edge 的扩展管理页
2. 开启“开发者模式”
3. 选择“加载已解压的扩展程序”
4. 选择当前项目目录 `spider_audio`

## 怎么用

1. 打开目标页面
2. 让页面把资源真正加载出来
3. 点击浏览器工具栏里的扩展图标
4. 在弹窗中查看 `视频 / 图片 / 音频 / 其他` 分类
5. 点击“下载”保存资源

建议在页面里主动做这些动作，能提高嗅探命中率：

- 播放音频或视频
- 打开图片预览
- 滚动页面触发懒加载
- 切换分页或素材列表

## 弹窗功能

- `抓取设置`：可以开关 `视频 / 图片 / 音频 / 其他`
- `钉住`：把资源面板固定到网页右上角
- `清空`：清除当前标签页已记录的资源
- 图片资源支持缩略图预览

## 下载说明

下载时扩展会优先尝试把资源转成浏览器可直接保存的数据再下载；如果失败，再回退到直接下载原始 URL。

因此对部分依赖当前页面上下文、Cookie 或来源页环境的资源，成功率会比直接复制链接下载更高一些，但不保证全部站点都有效。

## 适合的使用场景

- 下载当前页面已经展示出来的素材
- 快速定位网页里真实使用的音频、视频、图片 URL
- 分析一个页面到底加载了哪些媒体资源
- 提取站内预览页、详情页、播放页已经实际请求过的文件

## 不适合的使用场景

- 全站批量抓取
- 绕过 DRM 或其他强保护机制
- 对未加载资源做枚举扫描
- 去除服务端已经写进图片或视频中的水印

## 已知说明

当前代码实现里没有这些功能：

- `cURL` 解析
- 一键复制 `curl` 命令
- 下载历史列表

如果你在旧版 README 里看到这些描述，以当前代码为准。

## 开发说明

项目结构很简单：

- `manifest.json`：扩展声明
- `background.js`：后台嗅探、资源记录、下载逻辑
- `content.js`：页面内 hook、补充扫描、右上角悬浮面板
- `popup.html` / `popup.js`：弹窗界面和交互

## 使用边界

请仅对你有权访问、下载和使用的资源使用本工具，并自行遵守目标网站的服务条款、版权要求和相关法律规范。
