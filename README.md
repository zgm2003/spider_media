# 媒体嗅探下载器

一个基于 Chrome Manifest V3 的浏览器扩展，用来捕获当前页面实际加载过的媒体资源，并提供查看、分析和下载能力。

这个项目是“当前页媒体助手”，不是全站爬虫。

## 当前能做什么

- 监听当前标签页里的音频、视频、图片和其他可下载资源。
- 结合 `webRequest`、DOM 扫描和主世界桥接，提高资源捕获成功率。
- 在弹窗里按类型展示捕获结果。
- 对普通直链文件和 `blob:` 资源执行下载。
- 识别 HLS 和 DASH 流媒体索引文件。
- 按需分析 `m3u8` 和 `mpd` 内容。
- 为流媒体生成可直接使用的 `ffmpeg` 命令。
- 支持把资源面板固定到页面里，边浏览边操作。

## 项目结构

- `manifest.json`
  扩展清单文件。
- `shared.js`
  共享规则层，负责媒体识别、下载模式判断、消息协议和通用工具函数。
- `background.js`
  Service Worker。负责按标签页维护状态、执行下载、分析流媒体索引、向界面推送更新。
- `page-bridge.js`
  运行在页面主世界，把页面自身的请求和媒体线索转发给扩展。
- `content.js`
  负责 DOM 扫描、页面内浮层、以及 `blob` / 文本抓取兜底。
- `popup.html`
- `popup.js`
  弹窗界面和交互逻辑。

## 流媒体工作流

当扩展识别到 HLS 或 DASH 索引文件时：

- 普通的“下载”按钮仍然会下载索引文件本身。
- 弹窗里的“分析”按钮会解析索引内容。
- 弹窗里的“复制 ffmpeg”按钮会生成适合 PowerShell 使用的命令，例如：

```powershell
ffmpeg -user_agent "Mozilla/5.0 ..." -referer "https://example.com" -headers "Origin: https://example.com" -i "https://cdn.example.com/video.m3u8" -c copy "video.mp4"
```

这才是当前版本面对真实流媒体时的推荐用法。插件本身还不会自动把 HLS / DASH 合并成最终视频。

## 安装方式

1. 打开 Chrome 或 Edge 的扩展管理页面。
2. 开启开发者模式。
3. 点击“加载已解压的扩展程序”。
4. 选择当前项目目录。

## 使用方式

1. 打开目标页面。
2. 让页面把你关心的资源真正加载出来。
3. 点击浏览器工具栏中的扩展图标。
4. 按分类查看当前捕获到的资源。
5. 普通直链资源直接点击“下载”。
6. 遇到 `m3u8` 或 `mpd` 时，先点击“分析”，再点击“复制 ffmpeg”。

## 当前限制

- 不是全站爬虫。
- 不是 DRM 绕过工具。
- 不能去除水印。
- 还不能自动把 HLS / DASH 索引合并成最终视频文件。
- 捕获结果依赖当前页面实际加载或请求过的资源。
- 某些站点仍然依赖 Cookie、Referer 或页面上下文，离开页面环境后可能失败。

## 下一步建议

- 把 `<all_urls>` 权限收紧成按站点授权或白名单模式。
- 增加最小化的自动化冒烟测试，覆盖弹窗渲染和流媒体分析。
- 增加下载队列、重试和下载历史。
- 在通用链路稳定后，再补站点专用适配。

## 法律与使用边界

请仅对你有权访问、下载和使用的资源使用本工具。你需要自行遵守目标网站的服务条款、版权要求和相关法律法规。
